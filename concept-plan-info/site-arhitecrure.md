# Архітектура сайту — Сама Ліпила (концепт-редизайн)

> **Дата:** 01.07.2026
> **Статус:** v4 — описує **фактичний стан коду** (не план). Реструктуровано на основі аналізу всього проєкту.
> **Що змінилось із v3:** кошик, локації та контакти вже реалізовані (у v3 були в «ще не описано»). Планування/«прийняті рішення» винесено в кінець як історичний контекст.

---

## 1. Загальний огляд

- **Тип:** статичний сайт (SSG) — концептуальний редизайн реального бренду замороженних напівфабрикатів «Сама Ліпила» (Львів + Вінниця). Навчальний/портфоліо-проєкт, **не** офіційний сайт.
- **Стек:** Astro 6 + Tailwind v4 (конфіг у CSS, `@theme` у `global.css`), TypeScript. Шрифти self-hosted (Lora — заголовки, Inter — тіло).
- **Деплой:** GitHub Pages, `base: "/sama-lypyla-concept/"`. Через це **всі внутрішні лінки йдуть через `import.meta.env.BASE_URL`**.
- **Індексація:** вимкнена (`noindex, nofollow` у `Layout.astro`) — щоб концепт не плутали з офіційним брендом.
- **Джерела правди по даних:**
  - `src/data/products.ts` — товари + категорії (8 категорій, 52 позиції).
  - `src/data/locations.ts` — 23 точки самовивозу (5 Львів + 18 Вінниця).
  - `src/data/nav.ts` — пункти головної навігації.

## 2. Глобальні елементи (на всіх сторінках)

Обгортка — `layouts/Layout.astro`: `<html>`, SEO/OG-мета, шрифти, `Header`, `<slot/>`, `Footer`, підключення `menu.js` + `cart.js`.

Липкий (`sticky top-0`), напівпрозорий фон.

- **По центру/справа (десктоп):** навігація `Desktop.Nav.astro`, яка рендерить пункти з `data/nav.ts`:

  | Пункт | Веде на | Примітка |

  | Головна | `/` | |
  | Як ми робимо | `/#USP` | якір на секцію USP головної |
  | Меню | `/catalog/` | лендинг каталогу |
  | Локації | `/locations/` | адреси + зони доставки |

- **Іконка кошика** (`/cart/`) з лічильником `#cart-count` — з'являється, коли в кошику ≥1 позиції (оновлюється з `cart.js` через подію `cart-updated` + `storage`).
- **Бургер** (`BurgerButton`) на мобільних → відкриває `BurgerPanel` (drawer), керується `scripts/menu.js` (клас `is-open`, закриття по Esc / кліку по оверлею / кліку по лінку).

5. **Нижня смуга** — `© рік`, посилання на Політику конфіденційності (`/privacy/`), дисклеймер «концепт-редизайн / навчальний проєкт» + дисклеймер «відгуки — приклади, не реальні».

## 4. Головна (`/`)

Композиція з компонентів у `components/MainPageComponents/`. Порядок секцій **зверху вниз** (як у коді `index.astro`):

1. **Hero** (`Hero.astro`) — повноекранний фон-фото продукту із затемненням, лого на «медальйоні», H1 «Напівфабрикат може бути якісним», підзаголовок, 2 CTA: **«До меню»** (`/catalog/`) + **«Як ми робимо»** (якір `#USP`).
2. **Категорії** (`Categories.astro` з `title="Меню"`) — сітка **8 плиток 4×2** (`CategoriesItems.astro`), кожна → `/catalog/[category]/`. Дані з `products.ts` (`categories`).
3. **USP «Виробництво за склом»** (`USP.astro`, `id="USP"`) — головний диференціатор. 3 блоки з фото у форматі zig-zag: «Свіжі продукти. Без хитрощів», «Ліпимо при тобі», «Заморозили — чекаємо тебе».
4. **Популярне** (`Popular.astro`) — «Те, що беруть найчастіше». 4 карточки товарів через `getHits(4)` (товари з `isHit: true`, сортовані за `popularity`). Картка = `ProductCard.astro`. Лінк «Усе меню» → `/catalog/`.
5. **Як замовити** (`HowToMakeOrder.astro`) — 4 кроки: Обрати → Додати в кошик → Надіслати → Забрати.
6. **Відгуки** (`Reviews.astro`) — «Що пишуть наші покупці». 3 mock-відгуки (Олена/Львів, Дмитро/Вінниця, Ірина/Львів), 5★. Дисклеймер про mock — у футері.
7. **B2B-тизер** (`B2BTeaser.astro`) — «Співпраця з бізнесом» (синій фон `deep-blue`), 3 цифри-переваги, CTA «Обговорити поставки» → `/wholesale/`.
8. **Тизер локацій** (`LocationsTeaser.astro`) — «Заходь до нас», 2 картки міст (Львів 5 точок / Вінниця 18) → якорі `/locations/#lviv` та `/locations/#vinnytsia`, лінк «Усі локації».

> **Примітка.** Компонент `About.astro` існує у `MainPageComponents/`, але **наразі не підключений** до `index.astro` (окремого блоку «Про нас коротко» на головній зараз немає). Розповідь про бренд повністю несе секція USP.

## 5. Каталог

### 5.1 Лендинг `/catalog/` (`catalog/index.astro`)

- `PageHeader` — «Меню» + опис («8 категорій, понад 50 позицій»).
- `Categories` з `showHeader={false}` (щоб не дублювати заголовок) — та сама сітка 8 плиток.
- `Popular` — той самий блок популярного, що й на головній.

### 5.2 Сторінка категорії `/catalog/[category]/` (`catalog/[category].astro`)

8 статичних сторінок (по одній на `CategorySlug`), генеруються через `getStaticPaths`, який дістає `categoryName`, `products`, `tags` з `products.ts`.

Вміст:
- Хлібна крихта «← Усе меню» (`/catalog/`).
- Заголовок категорії + лічильник позицій.
- **Панель фільтрів** (тільки якщо в категорії є теги): кнопка «Усі» + кнопки за реально присутніми тегами (`getTagsForCategory`). Фільтр **клієнтський, без фреймворка**: інлайн-`<script>` перемикає клас `hidden` карточкам за `data-tags`, взаємовиключно (як радіо).
- **Сітка карток товарів** (`ProductsCard.astro`), `grid-cols-1 sm:grid-cols-2 lg:grid-cols-3`.

> Сортування, пошук і фільтр ціни **навмисно відсутні** (надлишкові для 5–8 позицій на категорію).

### 5.3 Картка товару (`CatalogCategoryComponents/ProductsCard.astro`)

- Адаптивне фото (`<Picture>`).
- **Бейджі** (максимум 2, пріоритет): `isHit` → «Хіт», `isNew` → «Новинка», далі найхарактерніший тег з `tags[]`.
- Назва, вага, ціна.
- **«Детальніше»** — `<details>` з описом (якщо є `description`).
- **Кнопка «+ в кошик»** — прив'язана до `data-product-id`, працює через `scripts/cart.js`.

## 6. Кошик і оформлення (`/cart/`)

Повністю реалізовано (`pages/cart.astro` + `scripts/cart.js`).

- **Порожній стан** (`#cart-empty`) — іконка + «Твій кошик поки порожній» + CTA на меню. Показується за замовчуванням (сервер не знає `localStorage`).
- **Заповнений стан** (`#cart-content`, заповнюється JS) — список позицій із мініатюрою, кількістю (± / видалити) + бічний підсумок «Ваше замовлення» з сумою і кнопкою «Оформити замовлення».
- **Діалог оформлення** (`<dialog>`, прогресивне покращення через `data-state`):
  - Стан `form`: контактні дані (ім'я, телефон з `pattern` `+380…`), спосіб отримання (самовивіз / доставка), місто (Львів / Вінниця), точка самовивозу (`<select>` заповнюється з `locations.ts` за містом) **або** адреса доставки, коментар.
  - Стан `preview`: згенероване текстове повідомлення замовлення + кнопки «Копіювати», «Viber (демо)», «Telegram (демо)» + дисклеймер «концепт-демо, замовлення не надсилається».
- **`scripts/cart.js`** — стан у `localStorage` (ключ `sama-lipyla-cart`), експортує `getCart`, `getCartCount`, `getCartTotal`, `addToCart`, `removeFromCart`, `updateQuantity`, `generateOrderMessage`, `buildViberLink`, `buildTelegramLink`. Диспатчить подію `cart-updated` (її слухають лічильник у Header і рендер кошика).

> Дані про точки/міста в діалозі беруться з того самого `locations.ts`, що й сторінка `/locations/` — списки не розходяться.

## 11. Наскрізні патерни

- **Лінки:** завжди `${import.meta.env.BASE_URL}...` (ніколи `/catalog` від кореня) — інакше 404 на GitHub Pages. Ліки з завершальним `/`, щоб уникнути 301.
- **Регістр імен файлів** в імпортах ассетів має точно збігатися (CI на Linux регістрозалежний).
- **Зображення:** `<Picture>` для адаптивних карток (`formats={['avif','webp']}`), `<Image>` для фіксованих (лого). Стиль внутрішнього `<img>` через `[&_img]:...`.
- **Клієнтський JS без фреймворка:** фільтри каталогу, мобільне меню (`menu.js`), кошик (`cart.js`) — через `data-*`-атрибути + класи (`is-open`, `is-active`, `hidden`).
- **Палітра:** токени `primary` / `accent-red` / `deep-blue` / `cream` / `warm` (не дефолтні `orange-*/gray-*`).
- **Брейкпоінти сітки:** каталог `lg:grid-cols-3`, головна `lg:grid-cols-4`.
