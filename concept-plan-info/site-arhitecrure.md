## ⚠️ Спірне питання: розташування блоку «Про нас»

**Моя рекомендація:** «Про нас» **НЕ** одразу після Hero, а нижче — після популярних позицій або перед «Як замовити».

**Аргументи:**
- Бізнес-задача головної = довести користувача до товару.
- Послідовність «Hero → одразу каталог» — найкоротший шлях.
- Користувач спочатку хоче побачити **що продаєте**, потім — **кому довіряти**. Не навпаки.
- Розповідь про бренд має сенс **після** того, як людина зацікавилась продуктом.

**Якщо все ж залишити після Hero** — обмежити блок 2–3 короткими тезами, без CTA. Не давати людині застрягати тут.

**Рішення за тобою.** У структурі нижче зафіксовано **мій варіант** (Про нас нижче). Якщо вибереш інакше — переставимо в v4.

## Глобальні елементи (на всіх сторінках)

- **Лого** + назва «Сама Ліпила» (зліва).

  1. Про нас
  2. Меню *(посилання на `/catalog`)*

### Footer

Дефолтний, єдиний для всіх сторінок. Зміст уточнити окремо (контакти, соцмережі, політика конфіденційності, дисклеймер концепт-редизайну).

1. Hero
2. Категорії (сітка 4×2)
3. **Виробництво за склом** *(новий — головний USP-блок)*

5. Як замовити

7. **Відгуки клієнтів

9. Локації (тизер)
10. Footer

### 1. Hero

- **Логотип** (вгорі).
- **Слоган** — вміщає головний USP «виробництво за склом». Робочі варіанти для тестування на дні 2:
  - «Напівфабрикат може бути якісним. Подивись як ми його робимо»
  - «Лепимо при тобі. Морозимо для тебе»
  - «Запас домашньої їжі — без секретів виробництва»
- **Фон:** велике апетитне фото одного з топ-продуктів (вареники / пельмені / млинці).
- **Одна основна CTA-кнопка:** «Переглянути меню» → `/catalog`
- *(Опційно)* другорядне посилання нижче кнопки: «Як замовити» — якірне посилання до секції 7 нижче.

> Без плиток категорій у Hero. Категорії одразу нижче в окремій секції.

### 2. Категорії — сітка 4×2

8 плиток, симетрична сітка (4 в ряд на десктопі, 2×4 на планшеті, 1×8 на мобілі).

Кожна плитка: фон-фото категорії + назва. Клік → `/catalog/[category]`.

Повний список — див. розділ «Категорії та асортимент» нижче.

### 3. Виробництво за склом *(головний USP-блок)*

Окрема сильна секція. Структура:

- **Заголовок:** «Виробництво за склом» або «Подивись, як ми робимо твою їжу».
- **Велике фото процесу** (mock з Pexels — пошук «pasta making», «dumplings preparation», «hand making food»).
- **3–4 коротких тезисні картки:**
  - Ручна робота
  - Контроль складу
  - Свіжість щодня
  - Прозорість процесу
- *(Опційно)* мікро-цифри: «N кг тіста на день», «X позицій на тиждень» — заглушки, реальні цифри підставимо коли матимемо.
- CTA «Подивитись більше» → `/about` *(на тиждню 9; поки заглушка або без CTA).*

Це твій головний диференціатор. У жодного з 4 проаналізованих конкурентів немає повноцінного блоку про прозорість виробництва. Втратити цей блок = втратити USP.

### 4. Популярні позиції

4–6 карточок реальних товарів (не категорій).

На карточці: фото, назва, вага, ціна, бейджі (1–2), кнопка «+ в кошик».

**Як визначається популярність:** у `products.ts` поле `popularity: number` хардкод. Сортуєш список товарів по цьому полю спадно — топ-4/6 виводяться сюди. Поле не відображається на UI, використовується лише для сортування.

Альтернативно — фільтр `isHit: true` показує тільки відмічені вручну позиції. Простіше, прозоріше.

### 5. Про нас (коротко)

1 параграф або 3 короткі тези. Що сказати:

- Компанія, яка робить **домашні, смачні, якісні напівфабрикати**.
- **Ручна робота.**
- **Все свіже**, контроль складу.

CTA «Дізнатись більше» → `/about` (на тиждень 9; поки що заглушка або без CTA).

> Розташування — див. «Спірне питання» вище.

### 6. Відгуки клієнтів *(новий блок)*

3–4 mock-відгуки. Формат картки:

- Ім'я (вигадане, перші імена: Олена, Ірина, Дмитро, Анна).
- Місто (Львів / Вінниця).
- Текст відгуку (2–3 речення).
- *(Опційно)* зірочки 5/5 або просто без оцінки.

**Дисклеймер:** у Footer + у README обов'язково позначити «Відгуки — mock-data для концепт-редизайну».

У food-ніші довіра критична — відгуки закривають базову тривогу «а що там у складі насправді». Без них сайт виглядає холодніше за конкурентів.

### 7. Як замовити

Інформаційний блок, 4 кроки:

1. **Обрати** — переглянути меню.
2. **Додати в кошик.**
3. **Надіслати** — оформити замовлення (через сайт, WhatsApp/Telegram або дзвінком).
4. **Забрати** — самовивіз або доставка.

Реалізація: 4 карточки в ряд, з іконкою + коротким текстом для кожного кроку.

### 8. B2B-тизер *(новий блок)*

Короткий блок, який готує ґрунт для `/wholesale` (тиждень 9).

Структура:

- **Заголовок:** «Для бізнесу та закладів».
- **Підзаголовок:** «Оптові поставки напівфабрикатів для ресторанів, кафе, магазинів».
- 2–3 коротких пункти переваг (наприклад: «Стабільні поставки», «Адаптована тара», «Особистий менеджер»).
- CTA «Дізнатись більше» → `/wholesale`.

> На тиждню 7 — `/wholesale` буде заглушкою з текстом «Сторінка готується. Зверніться [контакти]». Повна сторінка — на тиждню 9.

### 9. Локації *(тизер)*

Невелика секція в кінці головної.

- Заголовок «Ми у двох містах».
- 2 картки міст: **Львів** і **Вінниця**.
- Клік по місту → перехід на `/locations` (з якірним посиланням на відповідну секцію).

## Сторінка: Меню (`/catalog`)

> URL = `/catalog`. Видимий текст у навігації та заголовках = «Меню».

### Структура сторінки

1. **Header** дефолтний.
2. **Заголовок** — «Меню».
3. **Сітка з 8 категорій** — карточки як на головній, повний список. Клік → `/catalog/[category]`.
4. **Footer** дефолтний.

> Пошук прибрано — на сторінці лише 8 плиток, шукати немає чого.

### Сторінка категорії (`/catalog/[category]`)

Окрема сторінка для кожної з 8 категорій (генерується через `getStaticPaths`).

1. **Header** дефолтний.
2. **Заголовок категорії** (наприклад, «Вареники»).
3. **Панель фільтрів:**
   - **Теги** (мульті-вибір): пісне, дитяче, м'ясне, з сиром, гостре, хіт *(6 тегів, набір остаточний).*
   - **Сортування:** за популярністю (спадно) / за ціною ↑↓ / за замовчуванням.
4. **Сітка карточок товарів.**
5. **Footer** дефолтний.

> Прибрано: пошук, фільтр ціни (надлишково для 5–8 позицій), сортування «Новинки» (фільтр порожній, бо всі позиції додані одночасно).

### Карточка товару

- **Бейджі** (1–2 видимих, поверх фото або над назвою):
  - «ТОП» / «Хіт» — якщо `isHit: true`
  - «Новинка» — якщо `isNew: true`
  - Тег-бейдж: «Пісне», «Дитяче», «Гостре» — найхарактерніший тег з `tags[]`
- **Фото.**
- **Назва.**
- **Вага.**
- **Ціна.**
- **Кнопка «Детальніше»** *(якщо є `description`)* — розкриває `<details>` з повним описом.
- **Кнопка «+ в кошик».**

> Без лічильника лайків, без кнопки лайка.

**Бейджі — правила вибору:**

Якщо у товару є і `isHit`, і `isNew`, і кілька тегів — показуємо максимум 2 бейджа. Пріоритет:

1. `isHit: true` → бейдж «Хіт»
2. `isNew: true` → бейдж «Новинка»
3. Один найхарактерніший тег з `tags[]` (наприклад, «Пісне» або «Гостре»)

## Теги (для фільтра на сторінці категорії)

Фінальний набір — **6 тегів**:

| Тег | Призначення | Приклади позицій |
|---|---|---|
| **Пісне** | Без м'яса, молока, яєць | Вареники з картоплею, з капустою, з вишнею, піца грибна |
| **Дитяче** | М'яке, без гострого | Сирники, солодкі млинці, вареники з солоним сиром, пельмені з індичкою |
| **М'ясне** | З м'ясом | Пельмені, бендерики, чебуреки, м'ясопродукти, м'ясні млинці |
| **З сиром** | З сиром | Вареники з солоним сиром, хінкалі з сиром, сирники, запіканки, чебуреки з сиром |
| **Гостре** | З гострими спеціями | Піца «Діабло», ковбаски гриль (опційно) |
| **Хіт** | Найпопулярніше | 5–8 топових позицій по `isHit: true` |

> Технічно: у `products.ts` поле `tags?: string[]`. На сторінці категорії — JS-скрипт слухає кліки по фільтрах, читає вибрані теги і додає/прибирає клас `hidden` карточкам, де `data-tags` не містить вибраних. Без React, без бібліотек.

### Бейджі (окрема система — для відображення на карточках)

Не плутати з тегами. Бейджі — це **візуальні мітки** на карточці. Деякі бейджі збігаються з тегами за назвою, інші — окремі.

- **«Хіт»** — `isHit: true`
- **«Новинка»** — `isNew: true`
- **«Пісне» / «Дитяче» / «Гостре»** — як видимий тег (один з `tags[]`, найхарактерніший)

## Структура даних `products.ts`

```ts
import type { ImageMetadata } from "astro";

export type CategorySlug =
  | "varenuki"
  | "pelmeni"
  | "syrniki"
  | "mlyntsi-salt"
  | "mlyntsi-sweet"
  | "pizza"
  | "myasoprodukty"
  | "vypichka-myaso";

export type ProductTag =
  | "pisne"
  | "dytyache"
  | "myasne"
  | "z-syrom"
  | "gostre";

export interface Product {
  id: string;             // унікальний slug, наприклад "vareniki-potato"
  name: string;           // "Вареники з картоплею"
  category: CategorySlug;
  weight: string;         // "500 г"
  price: number;          // в гривнях, ціле число
  image: ImageMetadata;   // через import з src/assets/products/
  description?: string;   // показується через <details> на карточці
  tags?: ProductTag[];    // для фільтрації + видимий бейдж (один з них)
  popularity: number;     // базове число для сортування (не відображається)
  isHit?: boolean;        // бейдж «Хіт»
  isNew?: boolean;        // бейдж «Новинка»
}
```

> `popularity` залишилось як невидиме поле для сортування. На карточці не показуємо — використовуємо тільки для алгоритму. Бейдж «Хіт» — окремий ручний прапор `isHit`, не похідний від `popularity`.

## Що ще треба описати окремо

Поточна v3 покриває **головну + меню + каталог**. До завершення IA треба зафіксувати:

1. **Кошик** (`/cart` — день 5 тижня 7).
2. **Локації** (`/locations` — день 6 тижня 7). Має містити дві секції: «Де нас знайти» (адреси, графіки, мапи) + «Зони доставки» (карта зон + умови).
3. **Контакти** (`/contacts` — день 6).
4. **Про нас** (`/about` — тиждень 9).
5. **B2B / Опт** (`/wholesale` — тиждень 9). На тижню 7 — заглушка зі словами «Сторінка готується».
6. **404** (окрема сторінка).
