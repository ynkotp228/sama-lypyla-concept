# Специфікація сторінки `/about` — «Сама Ліпила»

> Призначення файлу: технічне завдання для верстки сторінки в Claude Code.
> Архітектура та тексти вже погоджені (тиждень 9, день 1). Тут — фінальна структура + готовий контент.

---

## 0. Загальний принцип

Сторінка складається з 4 секцій в чіткому порядку: **Hero → Б (принципи) → A (контраст) → В (локальність) + CTA**.

Не оверінжинирити. Нових дизайн-токенів не вводити. Нових фото не додавати (жанр «руки в тісті» вже використаний на головній, повторювати не треба). Якщо фото захочеться додати пізніше — тільки в реєстрі «готова страва на столі», не «процес».

Обгорнути весь контент сторінки в `<main>` (урок тижня 8, день 6 — на `[category].astro` це забули зробити одразу).

---

## 1. Hero — компонент `PageHeader.astro`

Використати той самий компонент, що вже стоїть на `/cart`, `/locations`, `/contacts`. Новий компонент НЕ створювати.

**Props:**

```
title: "Що насправді означає «Сама Ліпила»"
description: "Не просто назва. Ось шість речей, на яких ми не економимо."
```

---

## 2. Секція Б — «Принципи» (checklist, основний блок)

Семантика: `<section aria-labelledby="principles-heading">`, заголовок `<h2 id="principles-heading">`.

Заголовок секції: **Наші принципи**

6 пунктів списком. Кожен пункт — іконка (lucide) + текст. Іконки — орієнтовні, підбираються під наявний iconset проєкту (перевірити, які lucide-іконки вже імпортовані в проєкті, тримати той самий патерн імпорту).

| # | Текст | Іконка (орієнтовно) |
|---|---|---|
| 1 | Ліпимо руками, не штампуємо на конвеєрі | `hand` або `users` |
| 2 | М'ясо вдруге не заморожуємо | `snowflake-off` або `package` |
| 3 | Начинку робимо щодня, залишки з учора не докладаємо | `calendar` або `clock` |
| 4 | Не тримаємо готову продукцію місяцями в морозильній камері: лише свіжі партії | `refresh-cw` |
| 5 | Скільки грамів на етикетці, стільки й у пакунку: без води та льоду | `scale` |
| 6 | Склад можна прочитати за п'ять секунд: жодного рядка не доведеться гуглити | `file-text` або `eye` |

Верстка — grid або list, 2 колонки на desktop / 1 на mobile (типовий responsive-патерн проєкту, як у card-грідах каталогу).

---

## 3. Секція A — «Контраст із масовим виробництвом»

Семантика: `<section aria-labelledby="contrast-heading">`, короткий блок, БЕЗ списку — суцільний абзац.

Заголовок секції (візуально можна менший за Б, це не рівнозначний за вагою блок): **Чому не як у всіх**

Текст:

```
Масове виробництво оптимізує ціну. Ми оптимізуємо смак. Тому не тримаємо тонни на складі й не розтягуємо термін придатності консервантами.
```

Без називання конкурентів чи брендів — нейтральний тон.

Візуально можна оформити як card-блок (rounded-2xl border, кремовий фон) — той самий стиль, що `<aside>` із зоною доставки на `/locations`. Не новий токен, перевикористання наявного.

---

## 4. Секція В — «Локальність» + CTA

Семантика: `<section aria-labelledby="locality-heading">`.

Заголовок секції: **Де нас знайти**

Текст:

```
Ми працюємо тільки у Львові та Вінниці, а не по всій Україні. У кожному місті є кілька точок самовивозу, звідки можна забрати вареники чи пельмені додому.
```

CTA-кнопка/лінк одразу під текстом, веде на `/locations` (враховуючи `base`, як і всі внутрішні посилання проєкту):

```
Дивись, де ми поруч → Наші точки
```

Використати наявний Button-компонент/стиль, якщо такий є в проєкті (перевірити, як оформлені CTA на головній чи в Footer).

## 5. Технічні нотатки

- `<meta name="robots" content="noindex, nofollow">` — має підхопитися автоматично через Layout, як на всіх інших сторінках. Перевірити, що не задубльовано вручну.
- Мета title/description сторінки — за наявним патерном інших внутрішніх сторінок (`/locations`, `/contacts`), не хардкодити новий формат.
- Жодних тире (—) в тексті. Весь наданий контент вище вже без них — при можливих правках/доповненнях з боку Claude Code теж не використовувати.
- Alt-тексти для іконок — декоративні, `aria-hidden="true"` (як `🚚` на `/locations`), текст пункту сам по собі зрозумілий скрін-рідеру.
- Перевірити після верстки: DevTools Network 0×404, `npm run build` без warnings.

## 6. Що НЕ робити

- Не додавати нових фото в жанрі «руки в тісті» — вже є на головній.
- Не дублювати блок «Ліпимо при тобі» з головної сторінки.
- Не додавати сьомий пункт у checklist без окремого узгодження.
- Не вводити нові Tailwind-токени/кольори під цю сторінку.
